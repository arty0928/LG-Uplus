113P 참고
1. 컴포넌트와 인스턴스 속성 비교하기  
  1.1 실습 파일 링크를 복제한다. 
  1.2 사각형 선택하고 Alt+드래그해서 복제본을 만든다. . 
  1.3 복제본을 선택한 상태에서 Ctrl+Alt+K로 컴포넌트로 등록한다. 
  1.4 Component를 선택하고 Alt+드래그해서 2개의 복제본을 만들고 Instance1, 2로 이름을 변경한다. 
  1.5 좌측 페널에서 Component>Rectangle2을 선택하고 색상을 변경하면 복제본도 모두 색상이 변경되는것을 확인할 수 있다.
  1.6 좌측 페널에서 Component>Rectangle2을 선택하고 우측 페널에서 radios를 24로 변경한다. 
  1.7 Instance1의 Rectangle를 선택하고 색상을 변경하면 
      다른 Instance나, 컴포넌트에 영향을 주지 않는것을 확인하자 

2. 컴포넌트 확인하기 
  2.1 좌측 페널에서 Assets를 선택하고 현재 파일을 클릭하면 등록한 컴포넌트가 보인다.
  2.2 컴포넌트를 드레그해본다. 
  2.3 Component1의 색상을 변경하면 드래그한 컴포넌트도 함께 변경되는걸 확인할 수 있다. 
  
3. 컴포넌트와 인스턴스로 버튼 만들기 
  3.1 좌측 페널에서 pages +를 눌러 새로운 페이지를 생성하고 이름을 Component로 변경한다. 
  3.2 하단 페널에서 새로운 프레임(#, F)로 드래그해서 872*712의 새로운 프레임을 생성하고 
      BG라고 이름을 변경한다. 
  3.3 사각형 툴(ㅁ, R)로 200*60 크기의 사각형을 생성하고 우측 페널에서 Fill 메뉴의 색상을 
      #ffffff로 설정하고 Stroke에서 +를 눌러 옵션을 활성화 한다. 
      weight:2 색상을 선택한다. 
  3.4 하단에서 텍스트 툴(T, 단축키 T)를 누르고 사각형 안에 Button_01로 변경하고 
      크기를 20으로 설정하고 중앙에 배치한다. 
  3.5 사각형과 문자를 드래그로 선택하고 Ctrl+G를 눌러 그룹으로 만든다. 
  3.6 group을 선택하고 Alt+드래그로 복제하고 
      사각형을 선택해서 fill의 색상을 변경한다.       
  3.7 두 Group을 선택하고 Alt+드래그로 2세트를 더 복제해서 3세트를 만든다. 
  3.8 첫번째 set를 선택하고 Ctrl+Alt+K를 눌러 컴포넌트로 등록한다. 
      ==> 두 버튼이 한개의 컴포넌트로 등록된다.  
  3.9 두번째 set를 선택하고 우측 패널에 컴포넌트 v를 눌러 create multiple Component로 등록한다.
      ==> Group별로 컴포넌트로 등록된다.     
  3.10 Assets에서 등록된 컴포넌트들을 확인하기
  3.11 두개를 하나의 컴포넌트로 등록한 것은 delete 눌러서 삭제한다. 
  3.12 file로 돌아와서 각각 등록된 component의 이름을 
       Button/01, Button/02로 변경하면   Button이 group명이 되고 숫자가 컴포넌트 이름이된다.  
  3.13 Assets에서 search부분에 그룹명(button)을 입력하면 해당 그룹에 등록된 컴포넌트를 쉽게 검색할 수 있다. 
  
4. 인스턴스 변경 및 해제 
  4.1 좌측 페널에서 Page +를 눌러 새로운 페이지를 만들과 하단 페널에서 새로운 프레임(#, F)를 눌러 
      프레임을 생성한다. 
  4.2 Assets에서 Button을 검색하고 button들(01, 02)을 드래그해서 instance를 만든다.
  4.3 이름을 instance1, instance2로 변경한다.  
  4.3 인스턴스된 instance1> Rectangle 1을 선택하고 fill, stroke를 변경해 본다. 
  4.4 instance1을 선택하고 우측 페널의 컴포넌트이름(01) 옆에 ...을 선택해서 
      push changes to main component를 선택하면 메인 컴포넌트가 변경된다. (바람직하지는 않음)
  4.5 instance2를 선택하고 우측 페널의 컴포넌트이름(02) 옆에 ...을 선택해서
      Detach Instance를 선택하면 원본 컴포넌트와 연결이 해제되서 원본 컴포넌트를 변경해도 
      instance는 변화가 없다. 
      ==> 기존 컴포넌트를 확장해서 새로운 컴포넌트를 만들때 사용
